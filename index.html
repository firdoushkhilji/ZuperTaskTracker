📑 Contract Automation Updates

📜 Description:

If 'Pool Opening' is 'Yes', first job is 'Pool Opening'. If 'No', jobs start from the contract start. Users can add 'Pool Closing' job with a selected date.

🧩 Case 1: At Creation

  • 📅 If dates match, create "Pool Closing" job, update expiry, delete future jobs, deactivate contract.

🧩 Case 2: During Contract

  • 📅 User selects a date ≥ current date. Scheduler updates and deactivates contract.

📏 Edge Case: Closing Date Beyond Expiry

  • 📅 Update expiry, create remaining jobs, deactivate contract after new date.

🎯 Focus:

Automate Pool Opening and Closing, ensuring smooth scheduling and contract management.
